# JUnit Assertions Exercise

## Objective
To understand and write automated tests using various JUnit 5 Assertions to validate test results.

## Explanation
In automated testing, **Assertions** are the heart of a test. They compare the actual result of your code against the expected result. If they match, the test passes. If they don't, the test fails.

We explored the most common JUnit 5 assertions:
- `assertEquals(expected, actual)`: Verifies two values are identical.
- `assertTrue(condition)`: Verifies that a condition evaluates to true.
- `assertFalse(condition)`: Verifies that a condition evaluates to false.
- `assertNull(object)`: Verifies that an object reference is null.
- `assertNotNull(object)`: Verifies that an object reference is not null.

By adding these simple statements, we ensure our Java code behaves exactly as intended without needing manual inspection!
